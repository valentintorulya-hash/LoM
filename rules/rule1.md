# Правила использования MCP инструментов

Этот файл определяет стратегию выбора инструментов для текущего проекта. Агент обязан следовать этим правилам при каждом запросе.

## 1. Sequential Thinking (`sequential-thinking`)
*   **Когда использовать:**
    *   При решении сложных логических задач.
    *   При планировании архитектуры приложения или новой фичи.
    *   Когда запрос пользователя требует многоступенчатого анализа.
    *   При отладке сложных багов, где причина неочевидна.
*   **Зачем:** Чтобы структурировать процесс мышления, выдвигать гипотезы и корректировать их до написания кода.

## 2. Context7 (`context7`)
*   **Когда использовать:**
    *   Когда нужно найти актуальную документацию по библиотеке (Zustand, Vite, TypeScript и т.д.).
    *   Если вы не уверены в синтаксисе метода или API.
    *   Когда нужно найти примеры использования сторонних библиотек, для которых нет специализированных MCP.
*   **Зачем:** Чтобы избежать галлюцинаций и использовать актуальные версии API.

## 3. Shadcn UI (`shadcn`)
*   **Когда использовать:**
    *   Для создания базовых UI компонентов (кнопки, инпуты, диалоги, карточки).
    *   Для построения форм и панелей управления (dashboards).
    *   Когда требуется чистый, доступный (accessible) и предсказуемый дизайн.
*   **Зачем:** Это основа дизайн-системы. Используем его по умолчанию для функциональных элементов.

## 4. Magic UI (`magic-ui`)
*   **Когда использовать:**
    *   Для создания Landing Pages.
    *   Для Hero-секций, анимированных списков, Marquee эффектов.
    *   Когда пользователь просит "красивый", "современный" или "вау" эффект.
*   **Зачем:** Для создания впечатляющих визуальных эффектов и маркетинговых блоков.

## 5. Aceternity UI (`aceternity`)
*   **Когда использовать:**
    *   Для специфических визуальных блоков (например, Glowing Grid, Sparkles, 3D эффекты).
    *   Если Magic UI не имеет нужного компонента, но требуется высокий уровень визуальной проработки.
*   **Зачем:** Для добавления уникальных, художественных UI элементов.

## 6. Playwright (`playwright`)
*   **Когда использовать:**
    *   Для браузерной автоматизации, скриншотов, проверки UI, сбора референсов.
    *   Когда нужно воспроизвести поведение сайта в браузере.
*   **Зачем:** Реальное отображение страниц и проверка UI/UX.

## 7. Dev Browser (`dev-browser`)
*   **Когда использовать:**
    *   Для интерактивных браузерных сценариев (формы, логины, последовательные клики).
    *   Когда нужна более долговременная сессия браузера.
*   **Зачем:** Удобный контроль пошаговых сценариев в браузере.

## 8. Brave Search (`brave-search`)
*   **Когда использовать:**
    *   Для общего веб-поиска (референсы, геймдев-статьи, UI примеры).
*   **Зачем:** Быстрый широкий поиск по интернету.

## 9. Tavily (`tavily`)
*   **Когда использовать:**
    *   Для глубокого исследования темы (баланс, геймдизайн, алгоритмы, туториалы).
*   **Зачем:** Сбор и выжимка информации из нескольких источников.

## 10. PixelLab (`pixellab`)
*   **Когда использовать:**
    *   Для генерации пиксель-арта (персонажи, предметы, тайлы).
*   **Зачем:** Быстрое создание стилизованных ассетов для игры.

## 11. Filesystem MCP (`filesystem`)
*   **Когда использовать:**
    *   Для прямых файловых операций внутри разрешенных директорий проекта.
*   **Зачем:** Безопасная работа с файлами (чтение/запись/список).

## 12. Nanobanana Pro (`nanobanana`)
*   **Когда использовать:**
    *   Для качественных анимаций, интерактивных эффектов и UI-переходов.
*   **Зачем:** Добавлять «живость» и производственную полировку UI.
